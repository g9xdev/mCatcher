# Media Catcher — native helper

This companion program lets the Media Catcher Firefox extension record live
streams with **ffmpeg** instead of doing everything in the browser. That means:

- one clean `.mp4` with **video + audio muxed** (no manual merge),
- correct duration / seek bar,
- **streams straight to disk** (no browser memory limit),
- native speed.

The extension automatically uses the helper once it's installed; if it isn't,
the extension falls back to its built-in in-browser recorder.

## Requirements

- **Python 3** (from python.org) — runs the host.
- **ffmpeg** — the installer downloads it for you, or you can drop `ffmpeg.exe`
  next to these files.
- Firefox with the Media Catcher extension loaded.

## Install (Windows)

From this folder, in a normal (non-admin) PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File install.ps1
```

The installer:
1. finds `pythonw.exe` and writes `mc_host.bat` (a windowless launcher),
2. writes the native-messaging manifest `com.mediacatcher.host.json`,
3. registers it under `HKCU\Software\Mozilla\NativeMessagingHosts`,
4. ensures `ffmpeg.exe` is available (downloads it unless `-SkipFfmpeg`).

Then **restart Firefox**. Open the popup — the footer shows **⚡ helper** when
it's connected. Record as usual; Stop finalizes the file, Save moves it to your
Downloads folder, and closing the tab before Save discards it.

## Self-update

The helper can install a new build for you:

1. Drop the new `media_catcher-*.zip` (and optionally `media-catcher-host.zip`) into a
   folder — your **Downloads** by default.
2. In the extension's **Settings → Updates**, set the **Extension folder** (where
   Firefox loads the add-on from, e.g. `C:\Code\mCatcher\media-catcher`) once, then click
   **Check & install update**.
3. The helper compares the package's version to what's installed, and if newer:
   - overwrites the extension source folder,
   - refreshes these host files,
   - stages a persistent `{id}.xpi` in your Firefox profile's `extensions/` folder,
   - pops a dialog offering to **restart Firefox now (restoring your tabs)**.

**Auto-install:** tick **"Auto-install when a new package appears"** and the helper
registers interest in the folder via OS change notifications (`ReadDirectoryChangesW`)
— no polling. When a newer `media_catcher-*.zip` lands, it offers to install + restart.

**Reliability guardian:** installs are handed to `guardian.ps1` (spawned detached so
it survives the Firefox restart). It **backs up** the current extension + host +
staged XPI (keeps the last 3 snapshots), **applies** the update, **verifies** it
(manifest parses + version; `mc_host.py` compiles + `VERSION`; XPI staged), and on
any failure **reverts** to the backup and tells you what went wrong. The extension
and host packages are versioned independently; a same-version host package whose
code changed is detected by content hash and offered with a confirmation prompt.

**Persistence caveat:** a *temporary* add-on (about:debugging) does not survive a
restart. For "restart and it's back," run **Firefox Developer Edition / Nightly /
ESR** with `xpinstall.signatures.required = false` (in `about:config`) so the staged
unsigned XPI installs permanently. On regular Firefox, use the folder overwrite +
**about:debugging → Reload** instead of a restart.

## Uninstall

```powershell
powershell -ExecutionPolicy Bypass -File install.ps1 -Uninstall
```

## Files

| file | purpose |
|------|---------|
| `mc_host.py` | the native messaging host; drives ffmpeg |
| `install.ps1` | registers the host with Firefox; fetches ffmpeg |
| `com.mediacatcher.host.json` | native manifest (generated by the installer) |
| `mc_host.bat` | launcher (generated by the installer) |
| `test_host.py` | offline protocol/command tests (`py test_host.py`) |

## How it works

```
Firefox extension ──native messaging (stdio)──▶ mc_host.py ──▶ ffmpeg ──▶ %TEMP%\media-catcher\mc_<id>.mp4
                                                                              │  (Save)
                                                                              ▼
                                                                        ~\Downloads\<title>.mp4
```

The extension sends the video (and paired audio) playlist URLs plus the page's
Referer/User-Agent. ffmpeg records with `-c copy` into a fragmented mp4 (so an
interrupted capture is still playable) and finalizes when the extension sends
`stop` (a `q` on ffmpeg's stdin). No transcoding — it's a fast remux.

Where “Save” puts recordings (native helper only). Blank = your Downloads folder.

Filters out tip sounds, avatars, and preview clips. 0 = show everything. Never affects HLS/DASH streams.

Re-encodes each saved recording and download to the chosen codec, then deletes the original — but only keeps it if the result is actually smaller. AV1 compresses best; H.265 is faster and more compatible. A GPU encoder (NVENC / QSV / AMF) is used when available. Needs the native helper.

Popup panel

Adds a panel beside the media list for the downloads queue and casting. Off = the classic single-column popup.

Lists active and just-finished downloads from every tab, with speed, progress, and cancel — not only this tab's.

Adds a Cast button to direct video files and a Now-Casting transport (play/pause, seek, volume) in the panel. Works with DLNA-capable TVs (most LG / Samsung / Sony smart TVs) — the helper serves the video to the TV over your network. First cast to a TV: accept the permission prompt on the TV with its remote. Apple TVs appear in the picker but aren't castable yet (needs an upstream pyatv fix). HLS/DASH streams and YouTube aren't castable yet. Needs the native helper.

Updates

Media Catcher checks GitHub Releases for new versions of the extension and the native helper. The helper installs them — backing up, verifying, and reverting on failure — then offers to restart Firefox (restoring your tabs). Requires the native helper.

The helper checks GitHub Releases on startup and every few hours, and offers to install + restart when a newer build lands. With this off, use the button below to check on demand.

Advanced — running an unsigned build from source (developers)

Only needed if you load the extension from a source folder on Firefox Developer/Nightly. On a normal signed install, leave these blank — Firefox updates the add-on itself, and the helper still updates from GitHub.
